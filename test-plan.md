# Тест-план для сайта InMotion Hosting

## 1. Введение
**Цель тестирования**: Проверка корректной работы ключевых функций сайта [InMotion Hosting](https://www.inmotionhosting.com/), включая навигацию, адаптивность, взаимодействие с пользователем и API-запросы.

## 2. Область тестирования
Тестируем следующие разделы и функции сайта:
- Главная страница (загрузка, отображение, навигация);
- Раздел "Hosting" (проверка выбора тарифов, функциональность кнопок);
- Форма регистрации и входа в систему;
- Контактная форма и форма заказа услуги;
- API (если доступен публичный API);
- Мобильная версия (адаптивность, кросс-браузерность);
- Корректность ссылок и внутренних переходов.

## 3. Типы тестирования
- **Функциональное тестирование** – проверка работы всех ключевых функций сайта;
- **UI/UX-тестирование** – анализ удобства использования, визуального оформления и адаптивности;
- **Кросс-браузерное тестирование** – проверка отображения сайта в различных браузерах (Chrome, Firefox, Edge);
- **Тестирование API** – отправка запросов с помощью Postman, проверка корректности ответов;
- **Тестирование производительности** – проверка скорости загрузки страниц;
- **Регрессионное тестирование** – проверка работоспособности после изменений.

## 4. Инструменты
- **Postman** – для тестирования API;
- **Selenium + Python (PyTest)** – для автоматизированного тестирования UI;
- **Jira** – для управления багами и тест-кейсами;
- **Browser DevTools** – для анализа сетевых запросов и ошибок в консоли;
- **GitHub** – для хранения тест-кейсов и отчетов.

## 5. Риски и ограничения
- Доступ к API может быть ограничен (если требуется авторизация);
- Возможны изменения на сайте, влияющие на тест-кейсы;
- Кросс-браузерное тестирование ограничено основными браузерами.

## 6. Критерии завершения тестирования
- Все критические баги исправлены и протестированы;
- Функциональность сайта соответствует требованиям;
- UI/UX соответствует стандартам удобства использования;
- Отчеты о тестировании загружены в GitHub.

## 7. Ожидаемый результат
После проведения тестирования должны быть выявлены и устранены возможные дефекты, сайт должен корректно работать во всех заявленных сценариях.
